# 🔍 Finding Microsoft Graph API

## 📋 You're in the right place!

You're on the "Request API permissions" page. Microsoft Graph should be visible, but here's how to find it:

---

## 🔍 Method 1: Scroll to Top (Most Common)

**Microsoft Graph is usually at the VERY TOP of the list:**

1. **Scroll UP** to the top of the page
2. Look for **"Commonly used Microsoft APIs"** section
3. **Microsoft Graph** should be the **FIRST card** in that section
   - It has a **blue hexagonal icon**
   - Title: **"Microsoft Graph"**
   - Description mentions: "Office 365, Enterprise Mobility + Security, Windows 10..."

---

## 🔍 Method 2: Use Search (If Available)

1. Look for a **search box** at the top of the dialog
2. Type: **"Microsoft Graph"** or **"Graph"**
3. It should filter and show Microsoft Graph

---

## 🔍 Method 3: Check Tabs

Make sure you're on the **"Microsoft APIs"** tab (not "APIs my organization uses" or "My APIs")

---

## 📋 What Microsoft Graph Looks Like:

- **Icon:** Blue hexagonal/polygon shape
- **Title:** "Microsoft Graph" (bold)
- **Description:** "Take advantage of the tremendous amount of data in Office 365, Enterprise Mobility + Security, and Windows 10. Access Microsoft Entra ID, Excel, Intune, Outlook/Exchange, OneDrive, OneNote, SharePoint, Planner, and more through a single endpoint."

---

## ✅ Once You Find It:

1. **Click on the Microsoft Graph card**
2. You'll then see the permission selection screen
3. Choose "Delegated permissions"
4. Select: Calendars.ReadWrite, User.Read, offline_access

---

**Try scrolling to the very top of the list - Microsoft Graph should be there!**
